RFC: Multi Data Document Site Framework Options

Primary author(s): Dumisani Modeya

1. Overview​

As the Multi Data Engineering team, we have been tasked with creating a document website for the business, to provide customers with access to general product information, product usage, as well as API documentation.

2. Objectives​

The objectives of this document are to:

  1. Clarify the scope of work for this task.
  2. List the requirements for the document site, both functional and non-functional.
  3. Provide a list of some of the most relevant document site frameworks that can satisfy most or, preferably, all the requirements, along with an overview and list of pros and cons for each.
  4. Propose a specific document site framework to be used.

3. Scope​

The scope of this document includes the document site framework to be used, as well as how, where, and the related details of hosting the document site framework.

4. Functional Requirements​

  • Navigation pane supporting nested, expandable/collapsible items
  • Page ordering
  • Snippet embedding
  • Lightweight
  • Performant
  • Easy creation and editing in a widely used markup language, such as Markdown

5. Non-functional Requirements​

  • Free to use
  • Widely used, for the purposes of effective, free, community-based support

6. Alternatives Considered​

We considered several alternative solutions for the document site framework. Below are the most relevant options:

6.1 Docusaurus​

Overview: Docusaurus is an optimised site generator built in React, suitable for creating documentation websites, blogs, and marketing pages.

Pros:

  • Independent pipeline for development and release
  • Not tied to platform releases
  • Highly customisable navigation pane
  • Supports Markdown for content creation
  • Lightweight and performant
  • Free to use
  • Large, active community for support

Cons:

  • Requires knowledge of React for advanced customisation
  • Initial setup might be complex for beginners

6.2 MkDocs​

Overview: MkDocs is a static site generator that's geared towards project documentation. It is written in Python and uses Markdown for content creation.

Pros:

  • Simple and easy to set up
  • Supports Markdown natively
  • Fast and lightweight
  • Highly customisable with themes and plugins
  • Free to use and open-source
  • Large community support

Cons:

  • Limited in terms of advanced features compared to Docusaurus
  • Customisation requires knowledge of Python and Jinja2 templates

6.3 VuePress​

Overview: VuePress is a static site generator powered by Vue.js, designed for creating documentation sites.

Pros:

  • Simple setup and configuration
  • Supports Markdown with Vue components
  • High performance and lightweight
  • Built-in search functionality
  • Free to use and open-source
  • Strong community support

Cons:

  • Requires knowledge of Vue.js for advanced customisation
  • Smaller ecosystem compared to React-based solutions

6.4 GitBook​

Overview: GitBook is a modern documentation platform that allows you to create and publish documentation easily.

Pros:

  • User-friendly interface
  • Supports Markdown and rich media
  • Collaborative editing features
  • Built-in search functionality
  • Free tier available

Cons:

  • Limited customisation options on the free tier
  • Advanced features require a paid subscription
  • Less control over the hosting environment

6.5 Sphinx​

Overview: Sphinx is a documentation generator primarily used for Python projects but supports multiple languages.

Pros:

  • Highly extensible with plugins
  • Supports reStructuredText and Markdown
  • Excellent for API documentation
  • Free to use and open-source
  • Large community support

Cons:

  • Steeper learning curve, especially for non-Python developers
  • Requires knowledge of reStructuredText for full functionality

7. Proposed Solution​

After evaluating various frameworks and tools, Docusaurus stands out as the most suitable option for our new documentation site. Here’s why:

  • Independent Pipeline: Docusaurus supports independent pipelines for development and release, facilitating continuous integration and deployment.
  • Flexible Updates: Documentation updates are not tied to platform releases, allowing for more frequent and flexible content updates.
  • Navigation Pane: Docusaurus supports a highly customisable navigation pane with nested, expandable, and collapsible items.
  • Page Ordering: Pages can be easily ordered through configuration files.
  • Snippet Embedding: Supports embedding code snippets, making it easier to include reusable content.
  • IDE Integration: Compatible with popular IDEs, enhancing the content creation and editing experience.
  • Performance: Docusaurus is lightweight and performant, ensuring fast load times and a smooth user experience.
  • Markdown Support: Content can be written in Markdown, a widely used and easy-to-learn markup language.
  • Diagram Support: Supports diagrams using tools like Mermaid.
  • Cost: Docusaurus is free to use.
  • Community Support: It has a large, active community providing free support and contributions.
  • AWS Hosting: Can be easily hosted on AWS, ensuring reliability and scalability.

Instructions to Install Mermaid​

To ensure Mermaid diagrams work on the site, follow these steps:

  1. Install the Mermaid Plugin: bash npm install @docusaurus/theme-mermaid
  2. Configure Docusaurus to Use the Mermaid Plugin:

Update your docusaurus.config.ts file to include the Mermaid plugin.

import { Config } from '@docusaurus/types'; 
 
const config: Config = { 
  title: 'My Site', 
  tagline: 'Dinosaurs are cool', 
  url: 'https://your-docusaurus-test-site.com', 
  baseUrl: '/', 
  onBrokenLinks: 'throw', 
  onBrokenMarkdownLinks: 'warn', 
  favicon: 'img/favicon.ico', 
  organizationName: 'facebook', // Usually your GitHub org/user name. 
  projectName: 'docusaurus', // Usually your repo name. 
 
  presets: [ 
    [ 
      'classic', 
      { 
        docs: { 
          sidebarPath: require.resolve('./sidebars.ts'), 
          editUrl: 
            'https://github.com/facebook/docusaurus/edit/master/website/', 
        }, 
        blog: { 
          showReadingTime: true, 
          editUrl: 
            'https://github.com/facebook/docusaurus/edit/master/website/blog/', 
        }, 
        theme: { 
          customCss: require.resolve('./src/css/custom.css'), 
        }, 
      }, 
    ], 
  ], 
  themes: ['@docusaurus/theme-mermaid'], 
}; 
 
export default config;
  1. Add Mermaid Diagrams to Your Markdown Files:

You can add Mermaid diagrams to your Markdown files using the mermaid code block.

8. Cost​

As one of the non-functional requirements of the solution is free use, there are no cost considerations outside of the hosting costs for the site, which are included in the scope of this document.

9. Security, Privacy, and Compliance Considerations​

When implementing the proposed solution, the following security, privacy, and compliance considerations must be taken into account:

  • Data Security: Ensure that all data transmitted between the client and server is encrypted using HTTPS.
  • Access Control: Implement proper access control measures to restrict access to sensitive documentation.
  • Compliance: Ensure that the documentation site complies with relevant data protection regulations (e.g., GDPR, CCPA).

12. Glossary​

  • API: Application Programming Interface
  • CI/CD: Continuous Integration/Continuous Deployment
  • IDE: Integrated Development Environment
  • Markdown: A lightweight markup language with plain text formatting syntax
  • React: A JavaScript library for building user interfaces
  • Vue.js: A progressive JavaScript framework for building user interfaces
  • reStructuredText: A file format for textual data used primarily in the Python programming language community
